# YumMe 项目交接文档

> 最后更新：2026-07-20。当前产品方向以“直接制作精美数字手账”为准。

## 1. 产品目标

YumMe 不是无限画布工具，也不是连线知识图谱。主流程是：

1. 从素材盒选择正文、便签、引言、链接、数据、清单或时间线。
2. 在真实纸张式页面中直接查看构图、选择素材、编辑内容并调整顺序。
3. 点击按钮直接下载连续长页 PDF，不打开预览页或打印对话框。
4. 通过 DSL 让 AI 批量生成或增量修改手账素材。

持续判断标准：

- 编辑页面和导出 PDF 是否像真实手账或手抄报，而不是通用卡片网格。
- 不同素材类型是否一眼可辨。
- 时间线、标签、图片、清单等内容是否有自己的排版语法。
- 桌面和移动端是否都能顺畅完成“添加、编辑、排序、导出、保存”。

## 2. 当前编辑器

主编辑器位于 `web/src/pages/Editor.tsx`，采用三部分制作台：

- 左侧 `JournalSidebar`：素材入口与页面内容顺序。
- 中间 `journal-page`：真实纸张式所见即所得页面，支持选择和拖拽排序。
- 右侧 `CardEditor`：抽屉式素材编辑器，不改变中间纸张尺寸。

主流程已经移除：

- 连线模式。
- 平移与缩放。
- 框选和小地图。
- 依赖坐标自由摆放的交互。

旧 `CanvasView`、`EdgesLayer`、`LayersPanel`、`Minimap` 等文件仍存在，仅用于历史代码兼容和后续清理参考，目前不被新 `Editor` 使用。

## 3. 素材与智能排版

共享规则在 `web/src/editor/presentation.ts`：

- `timelineBlockIndex`：按标题或年份内容识别时间线。
- `journalCardSize`：推断 `compact / standard / wide` 三种宽度。
- `sortJournalCards`：使用旧 `x/y` 字段保存阅读顺序。
- `reorderVisibleJournalCard`、`moveJournalCard`：页面拖拽和大纲按钮排序。
- `ensureBlocksForType`：切换类型时补齐必要内容块。

桌面页面拖拽使用独立浮层和等尺寸占位块。命中区域在拖起时冻结，卡片让位不会反过来改变当前目标，因此不会在相邻位置闪烁。松手后才写入历史。移动端为保证页面滚动，排序使用大纲中的上下移动按钮。

六种基础类型：

- `standard`：正文分区。
- `note`：胶带、横格纸、轻微旋转。
- `quote`：大字号引言与署名。
- `link`：票券缺口、域名和跳转视觉。
- `stat`：印章式大数字。
- `todo`：笔记本竖线、完成计数和可勾选事项。

时间线使用 `standard + list` 保持 DSL v1 兼容，但在编辑器和导出中渲染为独立历程票据。标签使用多种剪贴纸、荧光笔和印章形态，不再统一渲染成 pill。

## 4. 数据兼容

后端协议、DSL v1 和数据库仍保留：

- `cards[].x / y / w`。
- `edges`。
- 六种 `card_type`。
- 五种 block。

新编辑器只用坐标字段保存顺序，不展示连线。不要删除 `edges` 或直接迁移旧数据，除非已有明确的数据迁移与回滚方案。

## 5. 导出 PDF

导出器位于 `web/src/editor/exporter.ts`：

- `exportHTML` 生成自包含的隐藏渲染文档。
- 6 列智能网格，按共享尺寸规则布局。
- 按类型输出不同结构。
- 自动识别时间线。
- 标签、纸纹、胶带、印章和票券等视觉与编辑器对应。
- 仅导出 `visible` 素材。
- 用户内容区不放 YumMe Logo 或产品品牌文案。
- 使用 `html2canvas-pro + jsPDF` 在当前页面内生成 PDF 并直接下载。
- PDF 固定为一张连续长页，不使用 A4 分页，不打开新标签页或打印框。

修改类型渲染或排版规则时，必须同时检查编辑器和导出器，避免所见与所得分叉。验证 PDF 时用 `pdfinfo` 确认 `Pages: 1`，再用 `pdftoppm` 渲染 PNG 检查底部是否截断。

```bash

cd web && npx tsc -b --force
cd web && npm test
cd web && npm run lint
cd web && npm run build
cd server && go test ./...
```

开发账号：

- `typer / types123456`
- 示例地址通常为 `http://localhost:5173/mind/1`

浏览器验收至少覆盖：

- 桌面编辑器。
- 打开素材编辑抽屉。
- 390px 移动端。
- 拖动中占位块稳定且其他素材平滑让位。
- PDF 直接下载、单页、首尾完整。

## 7. 当前重点

已完成：

- 主流程从无限画布切换为手账制作台。
- 六种类型差异化呈现。
- 时间线识别与独立排版。
- 标签剪贴纸化。
- 智能宽度和顺序管理。
- 编辑器与导出器共享呈现规则。
- 稳定的虚拟落位拖拽。
- 素材盒使用七种结构化缩略图区分内容类型。
- 直接下载连续长页 PDF。
- 用户内容页眉移除 YumMe Logo。
- 移动端主操作收敛。
- 新用户示例和官网文案移除连线、缩放教学。

后续高价值方向：

- 增加整页模板或构图预设，但不要让模板覆盖用户内容。
- 给图片素材增加裁切、旋转和照片说明。
- 为旧画布组件制定删除或归档计划。
- 继续让编辑器视觉与导出 PDF 更严格一致。

## 8. 注意事项

- `doc/load` 用于初始加载并重置历史；导入使用 `doc/replace`，不能混用。
- 保存策略是 localStorage 即时写入，30 秒自动 PUT，手动保存立即同步。
- TypeScript 开启严格未使用检查。
- 不提交 `server/data.db`、构建产物、Playwright 临时文件、`.codegraph/` 或 `.cursor/`。
- 图标统一使用 `@phosphor-icons/react`。
- 页面可见文案避免使用 `—`、`–`。
